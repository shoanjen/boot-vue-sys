## 2. 背景与目标

### 2.1 背景
高校社团数量多、活动频繁，普遍存在成员信息分散、招新/入社流程不统一、活动报名统计低效、通知触达不稳定等问题。希望通过一个轻量的社团管理系统，提升社团运营效率与信息透明度，降低社联/老师的管理成本。

### 2.2 目标
- 为学生提供统一入口：浏览社团、申请入社、报名活动、查看通知与个人参与记录
- 为社团提供基础运营能力：社团信息维护、入社审核、成员管理、活动管理、公告发布
- 为平台管理者提供基础监管能力：社团审核/下架、内容风险控制（MVP以人工审核为主）

### 2.3 成功指标
- 7日内至少完成：社团创建≥10个、学生入社申请≥100次、活动报名≥200次
- 核心流程成功率：注册/登录成功率≥98%，入社申请提交成功率≥99%，活动报名成功率≥99%
- 通知触达：站内通知阅读率≥50%（MVP以站内为主）

## 3. 用户场景

### 3.1 目标用户
- 学生：想加入社团、参与活动、获取社团信息与通知
- 社团管理员（社长/负责人/干事）：负责社团运营与成员管理
- 平台管理员（社联/老师）：负责社团治理、审核与异常处理

### 3.2 用户故事
- 作为学生，我希望能浏览社团列表和详情，以便选择感兴趣的社团并申请加入。
- 作为社团管理员，我希望能审核入社申请和维护成员角色，以便高效管理社团人员。
- 作为社团管理员，我希望能发布活动并统计报名情况，以便组织线下/线上活动。
- 作为平台管理员，我希望能审核社团创建申请并处理违规内容，以便保障平台秩序。

### 3.3 使用场景
**场景1：学生加入社团**
- 触发条件：学生对某社团感兴趣
- 用户行为：查看社团详情 → 提交入社申请（填写理由/年级/联系方式） → 等待审核 → 审核通过后成为成员
- 预期结果：学生可在“我的社团”看到加入成功的社团，并可报名活动/接收公告

**场景2：社团发布活动并收集报名**
- 触发条件：社团需要组织活动
- 用户行为：创建活动（时间地点/人数上限/报名截止） → 发布 → 学生成员报名 → 管理员查看报名名单并导出
- 预期结果：报名过程可控（上限/截止），名单可查询与导出（MVP可CSV）

**场景3：平台管理员审核社团**
- 触发条件：新社团提交创建申请或存在举报
- 用户行为：查看待审核列表 → 查看社团资料 → 通过/驳回（填写原因）
- 预期结果：审核结果可追溯，社团状态在前台展示一致

### 4.2 功能详情

#### 4.2.1 注册/登录/退出（P0）

**需求描述：**
用户可通过账号密码完成注册与登录，登录后获取访问令牌；退出会清除令牌并返回登录页。

**交互流程：**
1. 用户进入登录页，输入账号与密码
2. 系统校验账号状态（是否禁用）
3. 登录成功后进入首页（社团广场），导航展示“我的”
4. 用户点击退出，清除登录态

**页面元素：**

| 元素 | 类型 | 说明 | 校验规则 |
|------|------|------|---------|
| 账号 | 输入框 | 手机号/邮箱/学号（任选其一） | 必填，格式校验 |
| 密码 | 密码框 | 用户密码 | 必填，8-20位 |
| 登录按钮 | 按钮 | 触发登录 | 表单校验通过可点 |
| 注册入口 | 链接 | 跳转注册 | - |

**异常处理：**
- 账号或密码错误：提示“账号或密码错误”
- 账号被禁用：提示“账号已被禁用，请联系管理员”
- 网络/服务异常：提示“服务异常，请稍后重试”

**埋点需求：**
- login_success：登录成功
- login_fail：登录失败（附失败原因码）

#### 4.2.2 社团列表/详情（P0）

**需求描述：**
提供社团广场，支持搜索与基础筛选；详情页展示社团信息、当前状态、公告与活动入口，未加入用户可发起入社申请。

**交互流程：**
1. 用户进入社团广场，浏览列表
2. 用户输入关键词或选择类型筛选
3. 点击社团卡片进入详情
4. 在详情页点击“申请加入”（若满足条件）

**页面元素：**

| 元素 | 类型 | 说明 | 校验规则 |
|------|------|------|---------|
| 搜索框 | 输入框 | 社团名/关键词 | 可选，最长50字 |
| 类型筛选 | 下拉/标签 | 如文体/学术/公益等 | 可选 |
| 社团卡片 | 卡片 | 名称/简介/人数/状态 | - |
| 申请加入 | 按钮 | 进入申请表单 | 未加入且社团可加入时展示 |

**异常处理：**
- 社团已下架/封禁：详情页提示“社团不可用”，隐藏入社与报名入口

**埋点需求：**
- club_view_list：进入列表
- club_view_detail：查看详情
- club_search：搜索/筛选提交

#### 4.2.3 创建社团申请与平台审核（P0）

**需求描述：**
学生可发起“创建社团”申请，填写必要资料；平台管理员审核通过后社团上架，驳回需填写原因并通知申请人。

**交互流程：**
1. 发起创建 → 填写社团资料 → 提交
2. 平台管理员在后台待审核列表查看详情
3. 通过：社团状态变为“已上架”，申请人自动成为社团管理员
4. 驳回：社团状态为“已驳回”，申请人收到站内通知并可修改后再次提交

**页面元素：**

| 元素 | 类型 | 说明 | 校验规则 |
|------|------|------|---------|
| 社团名称 | 输入框 | 社团名称 | 必填，2-30字，唯一 |
| 社团类型 | 下拉 | 分类 | 必填 |
| 社团简介 | 文本域 | 简介与定位 | 必填，10-500字 |
| 指导老师 | 输入框 | 可选 | 可选，最长30字 |
| 联系方式 | 输入框 | 手机/邮箱/微信等 | 必填，最长50字 |
| 证明材料 | 上传 | 可选上传图片/文件 | P1，MVP可先不做上传 |

**异常处理：**
- 社团名称重复：提示“社团名称已存在”
- 审核驳回：展示驳回原因并支持再次提交

**埋点需求：**
- club_apply_submit：提交创建申请
- club_review_pass / club_review_reject：审核结果

#### 4.2.4 入社申请与审核（P0）

**需求描述：**
用户在社团详情页提交入社申请；社团管理员在后台进行审核。审核通过后用户成为成员，且可报名活动与接收公告。

**交互流程：**
1. 学生点击“申请加入” → 填写申请理由 → 提交
2. 社团管理员在“入社申请”列表查看申请信息
3. 管理员通过/驳回（可填写备注）
4. 学生收到站内通知，并在“我的申请”查看状态

**页面元素：**

| 元素 | 类型 | 说明 | 校验规则 |
|------|------|------|---------|
| 申请理由 | 文本域 | 加入动机与自我介绍 | 必填，10-200字 |
| 年级/学院 | 下拉/输入 | 基础信息 | P1 |
| 提交按钮 | 按钮 | 提交申请 | 表单校验通过可点 |

**异常处理：**
- 重复申请：提示“已提交申请，请等待审核”
- 社团不可加入（停招/下架）：提示“当前不可申请加入”

**埋点需求：**
- join_apply_submit：提交入社申请
- join_apply_result：审核结果（通过/驳回）

#### 4.2.5 活动创建与报名（P0）

**需求描述：**
社团管理员可创建活动并发布；成员可在活动列表报名或取消报名。需支持人数上限与报名截止校验。

**交互流程：**
1. 管理员创建活动（标题、时间地点、人数上限、截止时间、说明）→ 发布
2. 学生在社团活动列表查看活动详情 → 点击报名
3. 系统校验：是否成员、是否已报名、是否超过上限、是否截止
4. 报名成功后在“我的报名”可查看与取消（若未截止/未开始，规则见下）

**页面元素：**

| 元素 | 类型 | 说明 | 校验规则 |
|------|------|------|---------|
| 活动标题 | 输入框 | 活动名称 | 必填，2-50字 |
| 开始/结束时间 | 时间选择 | 活动时间 | 必填，结束>开始 |
| 地点 | 输入框 | 线下地点/线上链接 | 必填，2-100字 |
| 人数上限 | 数字输入 | 0表示不限（可选规则） | 必填，≥0 |
| 报名截止 | 时间选择 | 截止时间 | 必填，≤开始时间 |
| 活动说明 | 文本域 | 详细说明 | 必填，10-1000字 |

**异常处理：**
- 非成员报名：提示“加入社团后可报名”
- 已报名：提示“你已报名该活动”
- 已满员：提示“报名人数已满”
- 已截止：提示“报名已截止”

**报名取消规则（MVP建议）：**
- 截止前可取消；截止后不可取消（如需例外，由管理员手动处理，P1）

**埋点需求：**
- activity_create：创建活动
- activity_publish：发布活动
- activity_signup / activity_cancel_signup：报名/取消报名

#### 4.2.6 公告发布与阅读（P0/P1）

**需求描述：**
社团管理员可发布公告；成员与访客（可选）可在社团详情查看公告列表与详情。已读状态为P1。

**交互流程：**
1. 管理员发布公告（标题、内容）→ 发布
2. 用户进入社团详情 → 公告列表 → 查看详情
3. （P1）查看详情后记录已读

**页面元素：**

| 元素 | 类型 | 说明 | 校验规则 |
|------|------|------|---------|
| 标题 | 输入框 | 公告标题 | 必填，2-50字 |
| 内容 | 富文本/文本域 | 公告内容 | 必填，10-3000字 |

**异常处理：**
- 内容违规：MVP不做自动识别，仅后台下架（P1）

**埋点需求：**
- announcement_publish：发布公告
- announcement_view：查看公告

## 5. 非功能需求

### 5.1 性能需求
- 列表查询（社团/活动/公告）首屏接口响应时间：P95 < 500ms（无复杂关联查询，分页）
- 单社团活动报名并发：支持同一活动同时报名≥50并发（通过后端事务/乐观锁保证不超卖）

### 5.2 安全需求
- 登录态：使用 Token（如JWT）与过期机制；敏感接口需鉴权
- 权限控制：至少区分学生/社团管理员/平台管理员，防止越权操作
- 输入校验：后端参数校验与基础防注入（优先使用框架校验与ORM能力）
- 审计追踪：对审核、成员移除、活动下架等关键操作记录操作人/时间/原因（MVP可写入操作日志表）

### 5.3 兼容性需求
- Web端：Chrome/Edge/Safari 最近2个大版本
- 分辨率：PC优先，兼容常见移动端浏览器（MVP可先PC端）

## 6. 界面原型（文字版）

### 6.1 页面流程图
登录/注册 → 社团广场 → 社团详情 →（申请入社/活动报名/公告阅读）→ 我的（我的社团/我的报名/我的申请）

社团管理员：我的社团 → 管理后台（入社审核/成员/活动/公告）

平台管理员：管理后台 → 社团审核列表 → 审核详情

### 6.2 关键页面
**社团广场：**
- 顶部搜索与筛选
- 社团卡片列表（名称、类型、成员数、状态）

**社团详情：**
- 社团简介、负责人信息、入社按钮（条件满足时）
- Tab：活动 / 公告 / 成员（成员列表可仅管理员可见，P1）

**活动详情：**
- 时间地点、人数与报名状态、报名按钮
- 管理员可见：报名名单入口

**我的：**
- 我的社团（列表）
- 我的报名（列表+状态）
- 我的申请（入社/创建社团）

**管理后台（社团管理员）：**
- 入社申请（列表+审核）
- 成员管理（列表+移除+设置管理员）
- 活动管理（CRUD+发布/下架+报名名单）
- 公告管理（发布+列表）

**管理后台（平台管理员）：**
- 社团审核（列表+通过/驳回+原因）
- 社团管理（上下架/禁用，P1）

## 7. 数据需求

### 7.1 数据埋点
- 用户侧：注册/登录、浏览社团、提交入社申请、活动报名/取消、公告阅读
- 管理侧：创建社团申请、审核通过/驳回、活动发布/下架、成员移除

### 7.2 数据报表（MVP建议）
- 社团维度：成员数、活动数、活动报名人数、公告阅读量
- 平台维度：新增社团申请数、审核通过率、入社申请量、活跃用户数（DAU/WAU简化）

### 7.3 核心数据实体（概念模型）
- 用户 User：id、账号、密码摘要、角色、状态、创建时间
- 社团 Club：id、名称、类型、简介、状态（待审/上架/驳回/下架）、创建人、负责人
- 社团成员 ClubMember：id、clubId、userId、成员角色、加入时间、状态
- 入社申请 JoinRequest：id、clubId、userId、理由、状态、审核人、审核时间、驳回原因
- 活动 Activity：id、clubId、标题、时间地点、人数上限、报名截止、状态、创建人
- 报名 ActivitySignup：id、activityId、userId、状态、报名时间、取消时间
- 公告 Announcement：id、clubId、标题、内容、状态、发布时间、发布人
- 通知 Notification：id、userId、类型、关联对象、内容摘要、已读标记（P1）
- 审核记录 AuditLog：id、对象类型、对象id、动作、原因、操作人、时间
